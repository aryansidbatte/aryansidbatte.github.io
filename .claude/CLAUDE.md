# Portfolio Site — Project Memory

## Who I Am
Computer Science: Computer Game Design B.S., recently graduated. Applying to SWE, fullstack, and AI-adjacent roles. This site is my primary tool for getting hired -- it needs to look exceptional and load fast.

## The Site
- **Purpose**: Showcase projects, skills, and personality to potential employers and recruiters
- **Goal**: Aid in getting hired across a range of technical roles
- **Audience**: Recruiters, hiring managers, senior engineers reviewing candidates
- **Tone**: Confident but not arrogant. Technical but design-aware. Fresh graduate with real skills and taste.

## Tech Stack
- **Framework**: Astro
- **Components**: 21st.dev (React islands where needed)
- **Styling**: Tailwind CSS
- **Deployment**: GitHub Pages via GitHub Actions
- **Version control**: Git with worktrees for parallel feature work

## Installed Skills — Use These
- `ui-ux-pro-max` — invoke for ALL frontend and visual work, no exceptions
- `brainstorming` — invoke before any new section or feature, ask questions first
- `writing-plans` + `executing-plans` — write a plan before touching code, always
- `subagent-driven-development` + `dispatching-parallel-agents` — use for parallel builds
- `using-git-worktrees` — use with parallel agents
- `test-driven-development` — for any interactive components or logic
- `requesting-code-review` + `receiving-code-review` — before merging anything
- `verification-before-completion` — always verify before declaring done
- `systematic-debugging` — methodical approach when something breaks
- `writing-skills` — for bio, project descriptions, and all copy
- `humanizer` — run on ALL copy before it goes into any page, no exceptions

## Connected Tools
- **21st.dev**: source of UI components

## Design Principles
- The site IS a design artifact. It must reflect a CS + Game Design background -- technical depth with visual polish
- Avoid generic AI aesthetics: no purple gradients, no Inter/Roboto, no cookie-cutter layouts
- Every visual decision should feel intentional and memorable
- Mobile first, always

- About / bio

## Project Cards Should Include
- Live demo link (GitHub Pages project site)
- GitHub repo link
- Tech stack used
- What problem it solves (not just what it does)
- Role I played if collaborative

## Workflow Rules
1. Never jump straight to code — brainstorm and plan first
2. Show plans in chunks for approval before executing
3. Write failing tests before writing implementation code
4. Run humanizer skill on all copy before it goes into any page
5. Verify work before declaring any task complete
6. Never commit anything -- I handle all commits myself just tell me if i need to
7. One feature per branch, PR before merge

## Copy and Voice
- First person, direct, no fluff
- Lead with impact not process ("reduced load time by 40%" not "I worked on optimizing")
- Avoid buzzwords: passionate, innovative, leverage, synergy
- Short sentences. Confident.

## What Good Looks Like
A recruiter opens this site on their phone and immediately understands who I am, what I have built, and why they should reach out — in under 10 seconds.
